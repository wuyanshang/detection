你是一个数据库表结构语义分析专家。请根据整张表的信息，推断每个字段的规范化含义与主体。

【为什么要做这一步】
字段中文名往往质量不高、信息隐含在表名和英文名里。同一个"姓名"，在投保人表里是"投保人姓名"，在员工表里是"员工姓名"。
必须结合整表上下文（表名 + 全部字段），先锁定"主体"，才能正确分类。

【表信息】
- 系统名称: {{systemName}}
- 系统描述: {{systemDesc}}
- 表英文名: {{tableName}}
- 表中文名: {{tableChnName}}

【字段清单】
{{columnList}}

【任务】
1. 推断整张表的主体/主题（tableSubject），如"投保人信息""员工信息""客户信息"。
2. 对每个字段，给出：
   - normalizedName: 规范化检索词。展开英文缩写、补全主体、剥离无意义单位/编号。
     例："姓名"→"投保人姓名"，"zjhm"→"投保人证件号码"。
   - subject: 该字段的主体（客户/投保人/员工/受益人/被保险人 等）。
   - attribute: 属性（姓名/证件号码/电话/地址 等）。
   - confidence: high / low。仅根据表名+字段清单能明确判断则 high；否则 low 并保留原义。

【约束】
- 只基于给定信息推断，禁止凭空发挥。拿不准就标 confidence=low、normalizedName 保留原始中文名。
- columnName 必须原样回填，用于结果对齐。

【输出格式】严格 JSON，不要输出其它文字：
{
  "tableSubject": "整表主体",
  "fields": [
    {"columnName": "xm", "normalizedName": "投保人姓名", "subject": "投保人", "attribute": "姓名", "confidence": "high"}
  ]
}
