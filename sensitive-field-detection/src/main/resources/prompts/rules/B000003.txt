## B000003 - 字段中文名是枚举取值、不承载字段含义

**前提**：本规则仅在 stage1 消歧失败时，且被列入”待判规则”时才检查。

**判断步骤（按顺序执行，前一步得出结论后立即停止）：**

**第一步（硬门）：field_cn 中是否存在维度词？**

维度词 = 能回答”这个字段存的是什么维度信息”的词，无论是抽象分类词（状态、类型、标志、方向）还是业务具体词（渠道、来源、方式、用途、等级）。

- **有维度词 → 立即输出”不报”，终止，禁止继续分析。**
  不得以”呈现形式以枚举为主””字段名侧重取值展示”等理由推翻此结论。
- 无维度词 → 进入第二步。

示例（有维度词，第一步即停止，不报）：
- `性别` / `婚姻状态` / `借贷方向` / `险种类型` / `代码类型`
- `渠道:1-app;2-小程序;3-后台`（”渠道”是维度词）
- `接口调用状态: 1=成功, 2=失败, 3=异常`（”状态”是维度词）

**第二步：field_cn 是否只描述”有哪些值”而无任何字段含义？**

- 是（纯枚举，无维度词）→ 报
- 否 → 不报

示例（无维度词，进入第二步，报）：
- `0-否，1-是` / `1-正常，2-异常` / `男` / `已婚`

**拿不准时默认不报。**
