## B000002-person - 字段中文名缺少人员主体限定

**前提**：本规则由代码门控（hitB2-person）路由，field_cn 已确认含人员属性类高风险词（姓名 / 编号 / 证件 / 地址 / 关系）。

**判定核心（功能判定，不靠词表枚举）**
只问一件事：在 field_cn / field_en / table_cn **三处任一**，该高风险词有没有被"指明归属主体"的成分限定——能否回答"这是**谁 / 哪个主体**的 姓名/证件/地址"。
- 三处任一出现主体 → **不报**；
- 三处都没有，高风险词只伴随下方"不算主体"的词、或独自出现 → **报**。

**判定方法**：依次看 field_cn、field_en、table_cn 三处——
- **机械短路（最先执行，不需推理）**：高风险词前若出现**以『人』或『员』结尾、且非技术词**的词（任意 "X人 / X员"，如 送审人、领取人、经办人、投诉人、坐席员…），即判主体已限定、**立即不报、停止分析**。此判定是字面模式匹配，**不得**再评估该词是否"标准 / 稳定 / 核心 / 唯一 / 是客户还是员工"。
- **任一处**出现指向"某类人 / 某个机构组织 / 某个业务对象"的词 → 立即停止、不报。
  （例：table_cn=`客户信息表` 已含"客户"，则其下 `证件类型` / `姓名` / `证件号码` 一律不报，不必 field_cn 自己再带主体。）
- 三处都为空或只剩"不算主体"的词 → 缺主体 → 报。

**主体 = 任何指向"谁的 / 哪个主体"的词**（功能定义，不是封闭清单；凡符合即算，无需出现在任何举例里）
- 指人：投保人/被保人/受益人/客户/用户/会员/申请人/联系人…，以及"动作/环节 + 人/员"（领取人/经办人/投诉人/报案人/坐席员…）。
- 指机构或经办渠道：机构/网点/代理/服务代理/agent…。
- field_en 同义亦可（CUSTOMER / USER / AGENT / INSURED / PAYEE / OPERATOR …）；table_cn 点明主体亦可。

**"不算主体"——这才是封闭清单**（清单之外、能指向"谁/哪个主体"的词一律算主体）
- 纯场景/动作词，且不带「人/员」后缀：投保、理赔、预约、核保、保全、变更…
- 技术/结构词：字段、信息、数据、记录、值、字符串值、参数值…
- 序号 / 助词 / 泛化词：1、2、A、的、相关、其他…

**硬约束（违反即误报）**
1. 主体只需"存在"：不要求是投保人/被保人/受益人这类核心角色，不要求唯一或精确。
2. 不得把已成立的主体降级——如把"客户/用户"说成场景词、把"代理/服务代理"说成渠道属性词、把"领取人/送审人"拆回"领取/送审"。以下翻案理由**一律无效**："不是标准/核心业务角色"、"临时/不稳定角色"、"主体不唯一"、"不知道是客户还是代理人还是员工"——只要有主体词存在就不报，本规则不追究它是哪一个。
3. table_cn 同时含技术场景词（OCR/识别/信息/报告）不否定其中的主体词。
4. 高风险词只出现在"取值列举/括号说明"里、字段主体是"值/字符串值"等通用值列时 → 不是该属性字段，不报。
5. 多值/多渠道拼接脏串只要含主体词即不报（其命名问题归命名规范类规则）。
6. 拿不准一律不报。

**示例（只示意判定方法，不是穷举）**
- 报：`姓名`（表=预约表）、裸 `证件号码` / `证件类型`——去掉高风险词后无任何主体。
- 不报：`客户姓名` / `投诉人姓名` / `服务代理姓名`（均含主体）、`字符串值（姓名、证件号…）`（姓名仅为取值列举）。
